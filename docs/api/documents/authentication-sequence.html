    Diagramme de Séquence - Authentification

    Vue d'ensemble

    Ce document décrit le flux d'authentification du service. Il existe deux modes d'authentification : Local (OAuth2 avec PKCE) et Remote (IAM distant).

    Mode Local (OAuth2 avec PKCE)

    Points clés:

    • PKCE (Proof Key for Code Exchange): Sécurise le flux OAuth2
      • code_verifier généré et stocké localement
      • code_challenge envoyé à l'autorisation
      • code_verifier transmis à l'échange de token
    • Token stocké localement pour les requêtes suivantes
    • Refresh automatique si token expiré

    Mode Remote (IAM Distant)

    %%{init:{"theme":"dark"}}%% sequenceDiagram actor User as Utilisateur participant App as Application participant Service as ServiceRemote participant IAM as Serveur IAM<br/>(Externe) participant API as API Backend 
    User-&gt;&gt;App: Accède à la page login

App-&gt;&gt;Service: checkAuthentication()

Service-&gt;&gt;IAM: Redirige utilisateur
Note over IAM: Portail d&#39;authentification

IAM-&gt;&gt;User: Prompt d&#39;authentification
User-&gt;&gt;IAM: Authentification

alt Authentification réussie
    IAM-&gt;&gt;App: Redirige vers /login&lt;br/&gt;?success=1
    Note over Service: Détecte success=1
    
    Service-&gt;&gt;API: GET /me
    API-&gt;&gt;Service: Profil utilisateur
    
    Service-&gt;&gt;API: GET /documents
    API-&gt;&gt;Service: Liste des documents
    
    Service-&gt;&gt;App: Émet &#39;service:user:loaded&#39;
    Service-&gt;&gt;App: Émet &#39;service:documents:loaded&#39;
    
    Service-&gt;&gt;App: isAuthenticated = true
    App-&gt;&gt;User: Affiche contenu
else Authentification échouée
    IAM-&gt;&gt;App: Redirige vers /login&lt;br/&gt;?success=0
    Note over Service: Détecte success=0
    
    Service-&gt;&gt;App: isAuthenticated = false

    • Redirection externe pour l'authentification
    • Fetch des ressources après retour réussi
    • Événements pour notification des données chargées
    • Pas de stockage local du token (géré côté serveur)

    Flux de Vérification de Session Silencieuse

    %%{init:{"theme":"dark"}}%% sequenceDiagram participant App as Application participant Service as ServiceLocal participant KeyCloak as IAM<br/>(KeyCloak) 
    App-&gt;&gt;Service: checkSession()

alt Token valide en localStorage
    Service-&gt;&gt;Service: Réutilise le token
    Service-&gt;&gt;App: return true
else Token expiré
    Service-&gt;&gt;Service: Génère codeVerifier
    
    Note over Service: Crée iframe invisible
    Service-&gt;&gt;KeyCloak: GET /auth&lt;br/&gt;+ prompt=none&lt;br/&gt;(iframe)
    
    alt Session IAM active
        KeyCloak-&gt;&gt;KeyCloak: Auto-connexion
        KeyCloak-&gt;&gt;Service: Retourne code
        
        Service-&gt;&gt;KeyCloak: POST /token&lt;br/&gt;(échange code)
        KeyCloak-&gt;&gt;Service: Nouveau token
        
        Service-&gt;&gt;Service: Stocke token
        Service-&gt;&gt;App: return true
    else Pas de session IAM
        KeyCloak-&gt;&gt;Service: Pas de session
        Note over Service: iframe fermée
        Service-&gt;&gt;App: return false
    end

    Callbacks et Événements

    useAuth Callbacks

    const { isAuthenticated, user, checkAuthentication } = useAuth({
      service,
      onLogin: () => { /* Utilisateur connecté */ },
      onLogout: () => { /* Utilisateur déconnecté */ },
      onError: (error) => { /* Erreur d'authentification */ }
    });

    Events (ServiceRemote)

    • service:user:loaded - Données utilisateur chargées
    • service:documents:loaded - Liste des documents chargée

    Configuration Requise

    Variables d'environnement (IamSettings)

    • IamUrl: URL du serveur IAM (KeyCloak)
    • IamRealm: Realm KeyCloak
    • IamClientId: ID client OAuth2
    • IamClientSecret: Secret client (mode serveur)
    • IamEntrepotApiUrl: API backend (mode local)
    • IamEntrepotApiUrlRemote: API distante (mode remote)
